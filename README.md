# chandana-vs-punith

A 30-day challenge to learn **AWS** and **Agentic AI** deeply — and to learn from each other's mistakes along the way.

## The Challenge

We've kicked off a 30-day deep-dive into two of the fastest-moving areas in tech today. Instead of both learning the same thing, we split the territory:

| Learner | Focus Area |
| --- | --- |
| **Punith** | Agentic AI |
| **Chandana** | AWS |

The goal isn't just to specialize — it's to build strong foundations in *each other's* domains too, so that by the end of 30 days we're both comfortable bridging cloud infrastructure and agentic systems.

## Why This Repo Exists

This repo is a public learning log. Expect:

- Daily notes, code, and experiments from both tracks
- Honest documentation of the **beginner mistakes** people commonly make when jumping into advanced topics
- Side-by-side comparisons of how AWS and Agentic AI concepts overlap (and where they don't)
- Resources, references, and reading lists we found genuinely useful

If you're starting your own journey into either of these fields, the mistakes log is probably the most valuable part — we're documenting the things we *wish* someone had warned us about.

## Structure

One folder per day. Inside each day, both of us have our own notes file for that day.

```
chandana-vs-punith/
├── day-1/
│   ├── README.md     # the day's topic + plan
│   ├── punith.md     # Agentic AI angle
│   └── chandana.md   # AWS angle
├── day-2/
│   └── ...
└── ...
```

Day 1 topic: [AWS Lambda & Python Lambda](./day-1/README.md).

## Tracking Progress

We're aiming for **30 consecutive days** of meaningful progress. Each day's commit should leave behind something a future learner can read and learn from — not just "studied today."

## Stay in Sync With Current Tech

Both AWS and Agentic AI are moving fast. Part of the challenge is staying aligned with what's actually being used in production *right now*, not just what was hot a year ago.
